# Tax Data Model

The tax subsystem models jurisdictional rules that can be synchronised from plugins or maintained manually. Understanding the schema ensures that provider implementations populate the correct fields and that downstream reports can reason about applicability windows and provenance.

## Core Entity: `TaxRule`
| Field | Type | Description |
| --- | --- | --- |
| `id` | `int` | Primary key generated by SQLModel. |
| `jurisdiction` | `str` | ISO-style code (`US-FED`, `US-CA`, `EU-IE`, `GLOBAL`) identifying the taxing authority. |
| `scope` | `str` | Domain of the rule (`corporate_income`, `vat`, `payroll`, etc.). |
| `expression` | `str` | JSONLogic or domain-specific expression describing applicability and rate calculation. |
| `rate` | ``Decimal`` \| ``None`` | Optional convenience field for flat-rate rules. |
| `valid_from` / `valid_to` | ``date`` \| ``None`` | Effective window for the rule. |
| `metadata` | ``dict[str, Any]`` | Arbitrary structured metadata (statute references, data source URLs, notes). |
| `source` | ``str`` \| ``None`` | Provenance identifier supplied by plugins. |

Example JSON payload returned by `GET /tax/rules`:
```json
{
  "id": 42,
  "jurisdiction": "EU-IE",
  "scope": "vat",
  "expression": "{\"type\": \"rate\", \"value\": 0.23}",
  "rate": "0.23",
  "valid_from": "2024-01-01",
  "valid_to": null,
  "metadata": {"source_url": "https://taxauthority.ie/rates"},
  "source": "oecd_stub"
}
```

## Loading Rules via Plugins
- Providers implement `upsert_rules()` and yield `TaxRule` instances (or dicts convertible to models).
- The `TaxService` persists each rule, updating existing rows based on `jurisdiction`, `scope`, and `source` keys.
- Audit metadata records which actor initiated the sync (API, CLI, scheduler).

## Usage Patterns
- **Reporting** – The reports router enriches ledger outputs with applicable tax rules for compliance summaries.
- **Forecasting** – Future enhancements will adjust forecasted cashflows by applying jurisdictional tax obligations per horizon.
- **What-if Analysis** – Expose hypothetical rules by POSTing to the tax API with `source="manual"` and short validity windows.

## Roadmap Enhancements
Planned improvements captured in `PLAN.md`:
- Strong typing for `expression` (JSONLogic schema validation at ingest time).
- Override precedence rules when multiple jurisdictions overlap (federal vs state vs local).
- Source provenance storage (statute references, revision identifiers) to support audit trails.
- Automated refreshers for major jurisdictions (OECD feeds, US IRS publications) with change detection.
